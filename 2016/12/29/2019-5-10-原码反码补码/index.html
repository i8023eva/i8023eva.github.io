1. 原码

0按8位来看
[+0]原 = 0　0000000
[-0]原 = 1　0000000
0的原码表示不唯一

2. 反码

[+0]反 = 0　0000000
[-0]反 = 1　1111111
0的反码表示也是不唯一的

3. 补码

[+0]补 = 0　0000000
[-0]补 = [-0]反 + 1 = 1,0000,0000
对于8位字长,进位的1溢出,我们只能看到8个0,这就和+0是一样的.

补码解决了0的表示不唯一的问题.

3.1 补码的算术运算

通过引进补码,可将减法运算转换为加法运算.

[X+Y]补 = [X]补 + [Y]补
[X-Y]补 = [X+(-Y)]补 = [X]补 + [-Y]补

例1　66 - 51 = 66 + (-51) = 15

用二进制补码运算 :
[+66]补 = [+66]原 = 01000010
[-51]原 = 1　0110011
[-51]补 = 1　1001101
[+66]补 + [-51]补 = 1,0000,1111
=15

例2　X = -52 = -0110100, Y = 116 = +1110100, 求 X + Y = ?

[X]原 = 10110100
[X]补 = 11001100
[Y]补 = 01110100
[X + Y]补 = 01000000
X + Y = +1000000

3.1.1 溢出问题

除法运算溢出时,产生除数为0中断
乘法运算无溢出问题

例3 若 : X = 0111 1000, Y = 0110 1001　则 : X + Y = 1110 0001

如果是有符号数,次高位向最高位有进位, 而最高位向前无进位,产生溢出(两个正数相加变成负数).
如果是无符号数,就不算溢出,因为最高位向更高位没有进位.

3.2 补码的说明

正数不变,负数即用模减去绝对值.

3.2.1 特殊数 10000000

对无符号数 :
(10000000)_B = 128
在原码中定义为 :
(10000000)_B = -0
在反码中定义为 :
(10000000)_B = -127
在补码中定义为 :
(10000000)_B = -128

3.2.2 关于模

说一个钟表的例子:
　将指针从5点拨到1点;
　两种拨法:
　　逆时针: 5 - 4 = 1;
　　顺时针: 5 + 8 = 12 + 1 = 1;
　这是模12:
　　5 - 4 = 5 + 8 → 8为4的补数
　　[-4]补 = 12 - 4 = 8

二进制下，有多少位数参加运算,模就是在 1 的后面加上多少个 0.

3.2.3 -128的补码

有符号 无符号 二进制补码
0 0 0000 0000
1 1 0000 0001
… … …
127 127 0111 1111

有符号的从这里开始变化

-128 128 1000 0000
-127 129 1000 0001
… … …
-1 255 1111 1111
0 0 0000 0000

补码没有+0/-0之分,所以这个特殊的数就用-128来表示.
就像钟表例子一样,这就是一个环,超出之后其实又从0开始计算超出的部分.

用模来算的话 :
八位二进制的模 : 1 0000 0000;
-128的绝对值 : 1 000 0000
-128的补码 : 10000 0000 - 1000 0000 = 1000 0000

负数的补码等于反码加一
=> 1 0000 0000 - 1000 0000
=> (1111 1111 - 0000 0001) - 1000 0000
=> 1111 1111 - 1000 0000 + 1
=> 0111 1111 + 1

负数 => 对补码再取补
